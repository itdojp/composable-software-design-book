# Terminology Guide

## Source Of Truth

The reader-facing glossary is the source of truth for definitions.
`project-management/term-base.csv` is the editorial source of truth for cross-language mapping and canonical wording choices.
`TERMS.yml` is the lightweight repository term map that must stay consistent with the term base.

## First Occurrence

- Spell out the canonical English term on first occurrence in a chapter.
- If the term is non-obvious, add a short appositive definition in the same paragraph.
- Reuse the same capitalization and spacing after the first occurrence.

## Abbreviations

- Introduce an abbreviation only if it appears multiple times in the same chapter.
- Define the abbreviation on first use.
- Do not create multiple abbreviations for the same term across chapters.

## Glossary And Term Base Roles

- Use the glossary for reader-facing definitions and cross-references.
- Use the term base for editorial consistency, translation control, and synonym decisions.
- Add a term to the glossary when the reader needs a definition.
- Add a term to the term base when editors need a canonical wording decision, even if the glossary definition is still brief.

## Translation-Aware Usage

- Prefer the canonical English term in the public manuscript, even when a Japanese draft used a different phrasing.
- Do not introduce ad hoc synonyms in English prose when the term base already defines a preferred form.
- If a chapter needs a local shorthand, keep the canonical term visible nearby.

## Consistency Rules

- The same concept must keep the same English label across prose, figures, captions, and chapter titles.
- Terms that remain untranslated in Japanese drafts must also remain untranslated in the English manuscript.
- When a terminology decision changes, update the glossary, term base, and any impacted chapter text in the same change set.
