# Example Verification: Review Checks

The running example uses a small review checklist to keep its claims testable.
It remains the concise review artifact even when the example also uses a lightweight traceability matrix.

## Verification Checks

- The artifact set is complete.
- The human approval gate is explicit.
- The policy check is named separately from the execution step.
- The diagram labels match the canonical terms in the prose.
- The implementation workflow references the same artifact names.
- The traceability matrix points each core claim back to specification, design, and implementation artifacts.
