# Example Spec: Acceptance Criteria

The running example is accepted when the following conditions hold.

## Acceptance Criteria

- The change request, review plan, decision packet, approved change, and acceptance evidence are explicit artifacts.
- Policy evaluation, evidence collection, and human approval remain distinct steps even when some of them run in parallel.
- The synchronization boundary before review and execution is explicit and reviewable.
- Effectful steps emit execution trace entries that remain linked to the same change identity and plan revision.
- The example can be referenced coherently from specification, design, verification, runtime, review, and implementation discussions.
